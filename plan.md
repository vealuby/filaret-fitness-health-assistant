# План расширения функциональности

## 1. Быстрый онбординг одним сообщением
- Ввести команду/кнопку «Быстрый старт».
- Пользователь отправляет свободный текст; через LLM/регулярки парсим ключевые поля (подъём, сон, рост/вес, работа, тренировки, аллергии, вода, цели).
- При отсутствии данных бот точечно уточняет.
- FSM-онбординг оставить в качестве подробного режима.

## 2. Цели как модули
- На этапе онбординга и в `/profile` показывать кнопки с целями (сон, энергия, снижение веса, набор мышц, вода, тренировки, лекарства, симптомы).
- Хранить включённые модули в БД (JSON/таблица).
- Использовать флаги для условия показа подсказок/напоминаний.

## 3. Улучшение LLM-взаимодействия
- Добавить в главное меню кнопку «Спросить ИИ».
- UX: бот ждёт текст, прикладывает дисклеймер, сохраняет историю вопросов.
- Команда `/ask` остаётся, но `/help` описывает формат.

## 4. План питания (локализация и детализация)
- Перевести названия приёмов пищи на русский.
- Расширить рекомендации: примеры блюд, примерные калории и БЖУ.
- Учитывать цели пользователя (например, при «снижение веса» — меньшая калорийность).

## 5. Модуль тренировок вне онбординга
- Добавить кнопку «Я был на тренировке».
- По нажатию: спросить тип тренировки, длительность, интенсивность, самочувствие.
- Использовать данные для корректировки планов питания/восстановления.

## 6. Персонализация, лекарства, симптомы
- Персонализация: расчёт суточной потребности (Миффлин-Сан Жеор), sleep debt трекинг, адаптивные советы.
- Модуль лекарств: команда `/meds`, хранение расписаний, напоминания «Принял/Пропустить».
- Модуль симптомов: команда `/symptoms`, сбор описания и риска, LLM-ответ с дисклеймером.

## 7. UI/меню
- Главное меню: «План на день», «Вода», «Спросить ИИ», «Я был на тренировке», «Настроить модули».
- Отдельное меню `/modules` для включения/отключения целей и функций.
